## Handoff Protocol Reference

### Required Handoff Sections

Every handoff document MUST include:

1. **Document ID**: Unique identifier (e.g., ARCH-PM-2025-12-26)
2. **Date**: ISO 8601 timestamp (YYYY-MM-DDTHH:MM:SS.sssZ)
3. **From Agent**: Source agent name and mode
4. **To Agent**: Target agent name and mode
5. **Context Summary**: Project state, epic/story context
6. **Task Specification**: User story, acceptance criteria
7. **Current Workflow Status**: Relevant status from governance documents
8. **References**: Links to related documents
9. **Next Agent Assignment**: Who should receive next handoff
10. **Acceptance Criteria**: Checklist for completion

### Handoff Naming Convention

Format: `{source-agent}-{target-agent}-{epic/story}-{date}.md`

Examples:
- `architect-to-pm-governance-fixes-2025-12-26.md`
- `bmad-master-to-dev-mvp-3-tool-execution-files-2025-12-25.md`
- `dev-to-code-reviewer-25-6-2025-12-24.md`

### Status Update Procedure

1. **Source Agent** creates handoff document
2. **Target Agent** reviews and accepts handoff
3. **Target Agent** updates status to IN_PROGRESS in:
   - HANDOFF-INDEX.md
   - bmm-workflow-status-consolidated.yaml
4. **Target Agent** completes work
5. **Target Agent** updates status to COMPLETED in:
   - HANDOFF-INDEX.md
   - bmm-workflow-status-consolidated.yaml
6. **Both Agents** update governance index
